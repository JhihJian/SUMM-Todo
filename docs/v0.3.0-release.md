# v0.3.0 Release Notes

This release introduces **Project Support**, allowing you to organize tasks into projects for better task management.

## What's New

### Project Management

Tasks can now be organized under projects. Each task belongs to at most one project.

**Project Commands:**

```bash
todo project add <name> [-d <description>]   # Create project
todo project edit <name> [-n <new_name>] [-d <desc>]  # Edit project
todo project list                            # List all projects
todo project show <name>                     # Show project details + stats + recent tasks
todo project delete <name>                   # Delete project (must clear tasks first)
```

### Task-Project Association

Add tasks to projects using the `<project>:` prefix syntax:

```bash
todo add "web-app: Implement login feature"
```

### Project Filtering

Filter tasks by project:

```bash
todo list -p <project_name>
```

### Project Grouping

Tasks are automatically grouped by project in the output:

```
=== web-app (3) ===
○ 5315eb92 [Setup database]
○ ef684e13 [Create API endpoints]
○ 7b545c54 [Build frontend]

=== (no project) (1) ===
○ f11736e2 [Standalone task]
```

## Database Changes

- New `projects` table with `id`, `name`, `description`, `created_at` columns
- `tasks` table now has optional `project_id` column
- Migration `v3.sql` handles schema upgrade automatically

## Error Handling

| Error Code | Description |
|------------|-------------|
| `E_PROJECT_NOT_FOUND` | Project does not exist |
| `E_PROJECT_EXISTS` | Project name already taken |
| `E_PROJECT_HAS_TASKS` | Cannot delete project with tasks |

## Breaking Changes

None. Existing tasks without projects continue to work as before.

## Upgrade

The database migration runs automatically on first use. No manual intervention required.

```bash
# Update via install script
curl -sSL https://raw.githubusercontent.com/JhihJian/SUMM-Todo/main/update.sh | bash

# Or download binary from Releases
```
