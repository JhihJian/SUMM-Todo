# v0.1.0 Release Notes

First stable release of Todo CLI - a Human-Agent Task Coordination Protocol.

## Core Features

### Task State Machine

Strict 5-state machine with enforced transitions:

```
                    ┌──────────┐
                    │cancelled │
                    └──────────┘
                         ↑
┌─────────┐  next/start  ┌─────────────┐  done   ┌──────┐
│ pending │ ───────────→ │ in_progress │ ──────→ │ done │
└─────────┘              └─────────────┘         └──────┘
                               ↑  │
                        resume │  │ block
                               │  ↓
                            ┌───────────┐
                            │  blocked  │
                            └───────────┘
```

**State Rules:**
- `pending` → `in_progress` (via `next` or `start`)
- `pending` → `cancelled`
- `in_progress` → `done` (requires `--result`)
- `in_progress` → `blocked` (requires `--reason`)
- `in_progress` → `cancelled`
- `blocked` → `in_progress` (via `resume`)
- `blocked` → `cancelled`
- `done` / `cancelled` are terminal (no further transitions)

### Task Model

```json
{
  "id": "019c",
  "title": "Implement JWT auth",
  "creator": "human",
  "created_at": "2026-02-25T10:00:00Z",
  "priority": "high",
  "tags": ["backend", "auth"],
  "parent_id": null,
  "due": null,
  "status": "pending",
  "assignee": null,
  "blocked_reason": null,
  "result": null,
  "artifacts": [],
  "log": null,
  "started_at": null,
  "finished_at": null
}
```

**Fields:**
- **Identity**: `id` (4-8 hex chars, auto-generated)
- **Immutable**: `title`, `creator`, `created_at`
- **Mutable**: `priority` (high/medium/low), `tags`, `parent_id`, `due`
- **State**: `status`, `assignee`, `blocked_reason`
- **Result**: `result`, `artifacts`, `log`, `started_at`, `finished_at`

### Output Format

**JSON (default)** - For Agent/AI consumption:
```bash
$ todo list
[
  {
    "id": "019c",
    "title": "Implement JWT auth",
    "status": "pending",
    ...
  }
]
```

**Pretty (`-p` flag)** - For humans:
```bash
$ todo -p list
○ ! 019c [Implement JWT auth] #backend #auth
○ · 01a2 [Update README] #docs
```

Status icons: `○` pending, `◐` in_progress, `⊘` blocked, `●` done, `✕` cancelled
Priority icons: `!` high, `·` medium, `_` low

### Error Handling

All errors output JSON to stderr with error codes:

```json
{"error": "E_INVALID_TRANSITION", "message": "Invalid state transition: cannot go from pending to done"}
```

**Error Codes:**
| Code | Description |
|------|-------------|
| `E_INVALID_TRANSITION` | Invalid state transition |
| `E_RESULT_REQUIRED` | `--result` required for `done` |
| `E_BLOCKED_REASON_REQUIRED` | `--reason` required for `block` |
| `E_TASK_NOT_FOUND` | Task ID not found |
| `E_QUEUE_EMPTY` | No pending tasks for `next` |
| `E_DATABASE` | Database error |
| `E_INVALID_INPUT` | Invalid user input |
| `E_PARSE_ERROR` | Parse error (date, enum, etc.) |
| `E_IO` | I/O error |

**Exit Codes:**
- `0` - Success
- `1` - User input error
- `2` - System/internal error

### Storage

- **Location**: `~/.todo/todo.db` (or `TODO_DB_PATH` env var)
- **Format**: SQLite with WAL mode
- **Schema**: Auto-migration on startup

### Time Parsing

Supports relative and absolute time formats:

```bash
# Due dates (future)
--due "2026-03-15"    # Absolute date
--due "3d"            # 3 days from now
--due "2w"            # 2 weeks from now

# Since filters (past)
--since "today"       # Start of today
--since "7d"          # 7 days ago
--since "2026-01-01"  # Absolute date
```

### ID Generation

- Time-ordered UUID v7, truncated to 4 hex chars
- Auto-expands to 6 or 8 chars on collision
- Example: `019c`, `019c90`, `019c90a1`

## Known Limitations

- Single-user only (no multi-user support)
- No semantic search (planned for v1.1)
- No Web UI (planned for v2.0)
- No multi-device sync (planned for v3.0)

## Agent Integration

The CLI is designed for AI agent consumption:
- JSON output by default
- Structured error codes
- Deterministic state machine
- Agent can create subtasks with `--creator=agent`

See [agent-integration.md](./agent-integration.md) for details.
