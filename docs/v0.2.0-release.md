# v0.2.0 Release Notes

This release introduces TOON (Token-Oriented Object Notation) as the default output format, providing significant token savings for LLM/AI agent consumption.

## What's New

### TOON Output Format

TOON is now the default output format, replacing JSON. This format is designed specifically for passing structured data to Large Language Models with significantly reduced token usage.

**Key Benefits:**
- **18-40% token savings** compared to JSON
- Human-readable key-value syntax
- Compact array notation with size hints
- Perfect for AI agent consumption

### Output Format Options

| Flag | Format | Use Case |
|------|--------|----------|
| (default) | TOON | LLM/AI agent consumption (most efficient) |
| `--json` | JSON | Backwards compatibility, tooling integration |
| `-p/--pretty` | Pretty | Human terminal output |

### Example Comparison

**TOON** (default - most token efficient):
```
id: "019c"
title: Implement JWT auth
creator: human
created_at: "2026-02-25T10:00:00Z"
priority: high
tags[2]: backend,auth
status: pending
```

**JSON** (`--json` flag):
```json
{
  "id": "019c",
  "title": "Implement JWT auth",
  "creator": "human",
  "created_at": "2026-02-25T10:00:00Z",
  "priority": "high",
  "tags": ["backend", "auth"],
  "status": "pending"
}
```

**Pretty** (`-p` flag):
```
○ ! 019c [Implement JWT auth] #backend #auth
```

## Technical Changes

### Dependencies

- Added `toon-format` v0.4 for TOON serialization

### API Changes

- `Output::new(pretty, toon, json)` - New constructor with format flags
- `Output::json()` - Helper for JSON mode (useful for testing)
- Error output now uses TOON format by default

### CLI Flags

- `--toon` - Explicit TOON output (same as default)
- `--json` - JSON output for backwards compatibility
- `-p/--pretty` - Human-readable output (unchanged)

## Migration Guide

### For AI Agents

No changes needed - TOON is the new default and is more token-efficient. Simply use commands as before:

```bash
todo list        # Now outputs TOON instead of JSON
todo show 019c   # TOON format
```

### For Scripts/Tooling

If you rely on JSON output, add the `--json` flag:

```bash
# Before (v0.1.0)
todo list | jq '.[] | .id'

# After (v0.2.0)
todo --json list | jq '.[] | .id'
```

### For Testing

Tests using JSON parsing should use `Output::json()` or the `--json` flag.

## Compatibility

- **Breaking Change**: Default output format changed from JSON to TOON
- **Mitigation**: Use `--json` flag for JSON output
- All other functionality remains unchanged
